
# Contributing

Pull requests are welcome!  
For major changes, please open an issue first to discuss what you would like to change.

## Workflow

1. Fork the repository  
2. Create a feature branch  
   ```bash
   git checkout -b feature-name
   ```
3. Make your changes  
4. Commit using [Conventional Commits](https://www.conventionalcommits.org/) style  
5. Push your branch and open a Pull Request

## Guidelines

- Keep commits small and meaningful  
- Follow existing code style  
- Update or add tests for any logic changes  
- Update `CHANGELOG.md` if relevant  
- Always verify your changes on Base Testnet before merging to main
